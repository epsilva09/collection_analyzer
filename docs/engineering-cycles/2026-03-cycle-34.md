## Goals

- Deliver the first comparison screen focused on A vs B overview metrics.
- Add progression-gap comparison blocks for myth, force wing, honor medal, and stellar.
- Add collection macro comparison block (completion/progress/reward tiers) in the same entry point.
- Add weighted PVE/PVP scoring with explainability in the same overview experience.
- Add selectable weight presets to adapt overview scoring strategy.
- Keep existing compare flows intact while introducing the new screen in parallel.

## Implemented Changes

### 1) Performance

- Reused cached Armory client wrappers from the foundation cycle, minimizing
  repeated remote requests per character comparison.
- Aggregated overview cards and progression rows in a dedicated service to keep
  view rendering lightweight.

### 2) Quality / Reliability

- Added `ArmoryCharacterNormalizer` to stabilize character profile payloads from
  `/api/website/armory?name=...`.
- Added `CompareOverviewService` with deterministic output shape for:
  - overview metric cards
  - weighted profile scores (PVE/PVP/overall) with contribution rows
  - selectable preset weights (`balanced`, `raid`, `duel`) with safe fallback
  - collection macro summary and top targets
  - collection reliability indicators aligned with controller/snapshot logic
  - progression-gap rows across systems
- Added focused tests for:
  - new Armory client character wrapper
  - compare overview service calculations
  - compare overview controller rendering

### 3) Architecture / Maintainability

- Added new route and controller action:
  - `GET /armory/compare/overview`
  - `ArmoriesController#compare_overview`
- Added new view:
  - `app/views/armories/compare_overview.html.erb`
- Added menu entry for the new comparison mode while preserving existing
  compare and compare-collections pages.

### 4) UX / Accessibility

- Introduced a dedicated comparison experience for fast decision making:
  - compact A vs B cards for key combat/profile metrics
  - weighted scoring cards (PVE/PVP/overall)
  - preset selector in compare form for score strategy
  - explainability tables by weighted metric contribution
  - collection macro cards (total/completed/in-progress/near completion)
  - explicit payload inconsistency counters (100% with pending materials)
  - collection priority ranking (impact x effort) for in-progress targets
  - collection diff chips (completed, average progress, near completion, reward tiers)
  - progression-gap table with per-system detail text
- Kept existing form pattern and i18n coverage in both Portuguese and English.

## Risks / Trade-offs

- Presets are static constants for now; user-defined/custom saved presets
  remain backlog for later iterations.
- The overview currently prioritizes core numeric comparisons; deeper stat
  explanations remain part of subsequent screens.
