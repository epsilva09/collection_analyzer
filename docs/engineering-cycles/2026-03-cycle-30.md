# Engineering Cycle Record — 2026-03 Cycle 30

## Goals

- Remove the narrow centered content pattern and better distribute page elements.
- Standardize filter and content layout across Armories pages.
- Improve readability, spacing, and sticky behavior consistency.
- Apply accessibility-focused refinements for keyboard/focus navigation.

## Implemented Changes

### 1) Performance

- Reduced layout thrash risk by consolidating responsive distribution in
  reusable CSS layout primitives instead of repeated page-specific overrides.
- Standardized sticky offset behavior via a single CSS variable
  (`--armory-sticky-offset`) to avoid per-view offset recalculations and
  visual drift.
- Added write debounce in `CollectionProgressTrackingService` to suppress
  unchanged snapshot writes captured inside a short interval window,
  reducing synchronous DB pressure under high request frequency.

### 2) Quality / Reliability

- Fixed a rendered text leak in Progress page caused by missing translation
  key in an ARIA label (`Heading">` symptom from `translation_missing`
  markup output).
- Corrected filter panel grid overflow in Progress (buttons escaping grid)
  by normalizing sidebar controls to vertical `col-12` structure.
- Preserved existing filtering behavior and Stimulus integration while changing only structural layout concerns.
- Hardened payload normalization in snapshot change classification to better
  handle legacy truthy flags (`"true"`, `"1"`, `1`) for
  `inconsistent_progress` and avoid false-positive diffs.
- Hardened `CollectionSnapshotService` parsing to ignore malformed
  `collections/rewards/missions` blocks without raising exceptions during
  snapshot generation.

### 3) Architecture / Maintainability

- Introduced reusable wide-layout primitives in `app/assets/stylesheets/armory.css`:
  - `.armory-layout-container`
  - `.armory-layout-main`
  - `.armory-workspace`
  - `.armory-workspace-filters`
  - `.armory-workspace-content`
- Added reusable compact sidebar mode:
  - `.armory-workspace-filters--compact`
- Migrated core views to the same layout pattern:
  - `app/views/armories/index.html.erb`
  - `app/views/armories/progress.html.erb`
  - `app/views/armories/progress_changes.html.erb`
  - `app/views/armories/materials.html.erb`
  - `app/views/armories/material_collections.html.erb`
  - `app/views/armories/compare.html.erb`
- Added tracking instrumentation event
  `collection_progress_tracking.write_throttled` for observability on
  debounced writes.
- Migrated project database stack from SQLite to PostgreSQL configuration:
  - `Gemfile` now uses `pg`
  - `config/database.yml` updated for PostgreSQL across environments
  - CI test job now provisions PostgreSQL service
  - Docker image includes PostgreSQL client/runtime libraries

### 4) UX / Accessibility

- Rebalanced page composition from single central column to wider workspace distribution.
- Standardized left-sidebar filter panel pattern across Progress,
  Materials, Material Collections, and Progress Changes.
- Improved sidebar visual density (spacing, labels, control heights) with compact mode.
- Unified sticky alignment so Materials and Progress filter/header elements remain at the same visual level.
- Added visible focus ring consistency for interactive controls inside Armory pages.
- Improved table readability in Progress Changes (column sizing, reduced overflow pressure, better Materials room).
- Added inconsistency badge in Progress and Progress Changes when API data
  reports progress/material mismatch (`inconsistent_progress`).
- Added optional toggle in Progress Changes to show stable pending materials
  (no quantity delta) for easier auditing.
- Added a dedicated collections-comparison screen (`A` vs `B`) with per-
  collection progress and bonus visibility.
- Added filters on collections-comparison (collection name, completion
  status, minimum progress delta) and visual bonus states:
  unlocked highlighted, locked bonuses displayed with lower opacity.

## Risks / Trade-offs

- The broader layout increases horizontal information density; while better
  for desktop scanning, it may require additional tuning for very small
  laptop widths.
- Sticky behavior now depends on a shared offset variable; future changes
  should update the variable rather than ad hoc `top` values.
