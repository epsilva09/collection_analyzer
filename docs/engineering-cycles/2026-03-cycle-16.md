## Goals

- Keep table headers fixed consistently.
- Remove dynamic JS offset logic that was causing visual drift.
- Simplify sticky behavior for easier predictability and maintenance.

## Implemented Changes

### 1) Performance

- Removed scroll/resize sticky offset recalculations from filters
  controllers, reducing event-driven recomputation.

### 2) Quality / Reliability

- Replaced variable sticky offsets with fixed `top` values in CSS for:
  - materials tables
  - material collections tables
  - progress table hook
- Prevented layout instability caused by runtime offset updates.

### 3) Architecture / Maintainability

- Removed dynamic sticky offset code paths from:
  - `materials_filters_controller.js`
  - `material_collections_filters_controller.js`
- Sticky header behavior is now declarative and centralized in CSS.

### 4) UX / Accessibility

- Headers stay fixed in a stable position while scrolling.
- Visual behavior no longer oscillates based on filter card height.

## Risks / Trade-offs

- Fixed top value (`0.75rem`) is globally consistent but less adaptive to
  future major layout height changes; can be tuned if UI structure changes.
