## Goals

- Replicate sticky-scroll filter behavior from Materials into Progress.
- Keep sticky calculations robust on resize and filter-state updates.
- Preserve current Progress UX (no structural changes in list rendering).

## Implemented Changes

### 1) Performance

- Sticky offset is calculated from the real filters card height and updated
  only on connect, apply, and resize events.

### 2) Quality / Reliability

- Added lifecycle-safe resize listener management in
  `progress_filters_controller.js`.
- Added deterministic CSS variable update for sticky offset in Progress.

### 3) Architecture / Maintainability

- Added dedicated `filtersCard` target in Progress view/controller,
  mirroring the Materials pattern.
- Added scoped `progress-data-table` sticky header CSS hook for
  consistency with existing sticky design primitives.

### 4) UX / Accessibility

- Progress filters now maintain stable sticky behavior while users scroll
  long results.
- Kept existing keyboard/filter interactions intact.

## Notes

- Progress currently renders collection entries as `list-group` blocks, not
  tables, so there is no rendered table header (`thead`) on this page to
  make sticky without changing the current UX structure.

## Risks / Trade-offs

- Sticky offset depends on computed card height; mitigated by recalculation
  on apply/resize.
