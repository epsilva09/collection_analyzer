## Goals

- Keep materials filters visible while scrolling down.
- Keep materials table headers visible during vertical scroll.
- Prevent overlap between sticky filters and sticky table headers.

## Implemented Changes

### 1) Performance

- Reused existing DOM cache and filter flow.
- Sticky offset is computed from current filters card height and updated on
  filter changes and window resize only.

### 2) Quality / Reliability

- Added deterministic sticky offset behavior in the Stimulus controller.
- Added resize listener lifecycle management (`connect`/`disconnect`).

### 3) Architecture / Maintainability

- Added dedicated class for sticky materials filters and table headers,
  avoiding side effects on other pages/tables.
- Added a specific table class to scope sticky header styles.

### 4) UX / Accessibility

- Filters now remain accessible while navigating long materials lists.
- Table column context remains visible via sticky header during scroll.

## Risks / Trade-offs

- Sticky stacking context depends on computed filter card height; behavior is
  mitigated with dynamic offset recalculation on input changes and resize.
